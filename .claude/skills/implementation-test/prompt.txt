Run tests for implemented changes.

No input required. Runs in current git branch.

Steps:
1. Run tests: pytest (config in pytest.ini: asyncio_mode = auto, testpaths = tests)
2. Capture output: test count, passed/failed
3. Check exit code: 0 = success, non-zero = failure
4. Report results:
   - If PASS: All X tests passed. Ready for verification.
   - If FAIL: X tests failed. List failed tests and error messages. Blocks workflow.

Output format - Success:
✅ All tests passed
- Tests: 45 passed
- Status: Ready for verification phase

Output format - Failure:
❌ Tests failed
- Tests: 45 collected, 3 failed
  - tests/test_chores.py::test_fetch_chores_returns_error failed: AssertionError: ...
  - tests/test_chores.py::test_load_chores_sets_error_state failed: ...
- Status: Blocked - fix tests and retry
